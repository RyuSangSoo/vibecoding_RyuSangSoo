# KineLab 작업계획서

## 목표

DH 파라미터, 정기구학, 직관적인 3D 시각화에 초점을 둔 Windows 친화적 교육용 매니퓰레이터 시뮬레이터를 만든다.

## 범위

### MVP 포함 범위

- 6축 매니퓰레이터 모델
- DH 파라미터 편집기
- 관절 각도 슬라이더
- 정기구학 계산 엔진
- 말단효과기 자세 표시
- 3D 프레임 및 링크 시각화
- 프리셋 로봇 구성

### MVP 제외 범위

- 완전한 강체 동역학
- 충돌 검출
- ROS 연동
- 고급 물리 시뮬레이션
- 다중 로봇 네트워킹

## 마일스톤

### 1단계: 프로젝트 기반 구성

- 저장소 구조 구성
- 앱 아키텍처 정의
- 프론트엔드 스택 선정
- 초기 문서 준비

### 2단계: 기구학 코어

- DH 변환 로직 구현
- 행렬 유틸리티 구현
- 관절 프레임 및 말단효과기 자세 계산
- 예제 6축 로봇 구성 추가

### 3단계: 시각화 시뮬레이터 초안

- 3D 매니퓰레이터 렌더링
- 관절 축과 기준 좌표계 표시
- UI 제어와 기구학 엔진 연결
- 실시간 자세 정보 표시

### 4단계: 교육 기능

- UI에서 각 DH 파라미터 설명 제공
- 프리셋 자세 추가
- 작업 공간 및 실습 예제 추가
- 수업용 가독성 향상

### 5단계: 패키징

- Windows 데스크톱 빌드 준비
- 사용 가이드 추가
- 스크린샷 및 데모 자산 추가

## 당장 진행할 작업

1. TypeScript + React + Three.js 애플리케이션 골격 구성
2. 재사용 가능한 기구학 모듈 구현
3. 실시간 슬라이더와 DH 테이블 입력이 연결된 초안 제작
4. 이후 IK와 과제 모드를 확장할 수 있도록 모듈화 유지

## 리스크

- 3D 렌더링을 초기에 과하게 설계하면 복잡도가 급격히 올라갈 수 있다.
- IK와 특이점 해석은 범위를 빠르게 키울 수 있다.
- 데스크톱 패키징은 시뮬레이터 초안이 안정된 뒤 진행하는 것이 좋다.

## 성공 기준

- 앱이 실행되고 6축 매니퓰레이터가 표시된다.
- 관절 값을 바꾸면 로봇 자세가 즉시 갱신된다.
- DH 파라미터를 수정하면 로봇 형상과 프레임 위치가 바뀐다.
- 말단효과기의 위치와 자세를 UI에서 확인할 수 있다.
