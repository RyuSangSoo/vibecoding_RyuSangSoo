# 교육용 매니퓰레이터 시뮬레이터 기획서

## 1. 프로젝트 개요

- 프로젝트명: `KineLab` (가칭)
- 한 줄 소개: Windows 환경에서 실행되는 교육용 6축 매니퓰레이터 시뮬레이터로, DH 파라미터와 정기구학/역기구학을 직관적으로 학습할 수 있는 실습 도구
- 대상: 로봇공학 입문 수업, 메카트로닉스 실습, 학부 기구학 교육, 동아리/경진대회 데모

## 2. 문제 정의

로봇공학 수업에서는 DH 파라미터, 좌표계 변환, 정기구학, 역기구학이 핵심 개념이지만 학생 입장에서는 수식만으로 구조를 이해하기 어렵다.

현재 널리 알려진 시뮬레이터는 다음과 같은 한계가 있다.

- Ubuntu 중심의 설치 및 의존성 환경이 필요한 경우가 많다.
- 교육용 UI보다 연구/개발용 기능에 초점이 맞춰져 있다.
- 수업 시간 안에 빠르게 설치하고 개념을 실습하기에 진입 장벽이 높다.

## 3. 목표

이 프로젝트의 목표는 다음과 같다.

- Windows PC에서 쉽게 실행되는 교육용 매니퓰레이터 시뮬레이터 제공
- 6축 로봇암을 통해 DH 파라미터와 좌표계 개념을 시각적으로 학습
- 학생이 관절값과 DH 파라미터를 직접 바꾸면서 결과를 즉시 확인
- 교수자/조교가 수업 자료와 과제를 배포할 수 있는 기반 마련

## 4. 핵심 제안

본 프로젝트는 범용 산업용 시뮬레이터를 재현하는 것이 아니라, 교육 목적에 최적화된 `기구학 중심 시뮬레이터`를 직접 구현한다.

핵심 방향은 다음과 같다.

- `UR 스타일`의 6축 매니퓰레이터를 기본 모델로 제공
- 각 링크와 관절의 프레임을 3D로 시각화
- DH 파라미터 테이블을 수정하면 로봇 구조와 말단 자세가 즉시 반영
- 관절 공간과 작업 공간을 함께 보여주어 개념 연결을 돕는다
- 실습 모드와 데모 모드를 분리해 수업 활용성을 높인다

## 5. 사용자 시나리오

### 학생

- 관절 슬라이더를 움직이며 각 링크가 어떻게 변하는지 관찰한다.
- DH 파라미터 값을 수정하며 프레임과 말단 위치 변화를 학습한다.
- 주어진 목표 자세에 대해 정기구학 결과를 확인한다.
- 확장 기능으로 역기구학 실습이나 경로 계획 실습을 수행한다.

### 교수자

- 수업 중 좌표계와 링크 변환 과정을 실시간으로 설명한다.
- 실습용 로봇 모델과 예제 자세를 저장해 배포한다.
- 과제 모드로 특정 목표 자세, 제한 조건, 문제 시나리오를 구성한다.

## 6. 핵심 기능

### MVP 기능

1. 6축 매니퓰레이터 3D 렌더링
2. 관절 각도 슬라이더 제어
3. DH 파라미터 테이블 편집
4. 정기구학 계산
5. 말단 위치/자세 실시간 표시
6. 각 프레임 축 시각화
7. 프리셋 자세 저장 및 불러오기

### 확장 기능

1. 역기구학 계산
2. Jacobian 기반 해석 시각화
3. Joint limit 및 singularity 경고
4. 경로 보간 및 애니메이션
5. 교수자 과제 모드
6. 학생 풀이 기록 저장
7. 여러 로봇 모델 지원

## 7. 차별화 포인트

- 물리 기반 충돌 시뮬레이션보다 `교육용 기구학 시각화`에 집중
- Windows에서 쉽게 접근 가능
- 수식과 3D 결과를 동시에 보여주는 학습 경험 제공
- 단순 데모가 아니라 실제 수업에서 활용 가능한 UI 설계
- 오픈소스 형태로 확장 가능한 구조 지향

## 8. 구현 방향

### 왜 직접 시뮬레이터를 만드는가

직접 구현의 장점은 다음과 같다.

- 교육 목적에 맞는 UX를 처음부터 설계할 수 있다.
- 불필요하게 큰 외부 시뮬레이터 의존성을 줄일 수 있다.
- DH 파라미터, 변환행렬, 프레임 시각화 같은 교육 요소를 중심으로 제품을 설계할 수 있다.
- 경진대회 관점에서 구현 기여도가 명확하다.

단, 범위를 무리하게 넓히지 않기 위해 `직접 구현`의 의미는 아래처럼 정의한다.

- 직접 구현 대상: 로봇 모델 구조, DH 기반 기구학 계산, 3D 시각화, 교육용 UI, 저장/불러오기
- 초기 제외 대상: 고급 물리엔진, 정밀 충돌 해석, ROS 연동, 대규모 산업 환경 시뮬레이션

## 9. 추천 기술 스택

### 1순위 추천

- 언어: `TypeScript`
- UI: `React`
- 3D 렌더링: `Three.js`
- 빌드: `Vite`
- 데스크톱 패키징: `Electron` 또는 `Tauri`

### 추천 이유

- 교육용 프로그램은 표, 슬라이더, 수식 패널, 상태창 등 UI 비중이 높다.
- Three.js로 매니퓰레이터와 좌표계를 충분히 직접 렌더링할 수 있다.
- TypeScript는 수학 로직과 UI 상태를 함께 관리하기 좋다.
- 웹 기반으로 개발하면 나중에 브라우저 버전과 Windows 앱 배포를 동시에 가져갈 수 있다.
- 시연과 배포가 쉬워 경진대회 데모에 유리하다.

### 대안 스택

- `C# + Unity`
  - 장점: 3D 제작이 쉽고 시각적 임팩트가 강함
  - 단점: 교육용 표/폼 UI와 수식 인터랙션은 웹 스택보다 무거울 수 있음

- `Python + PySide`
  - 장점: 수학/로봇공학 라이브러리 친화적
  - 단점: 3D 인터랙션과 현대적 UI 완성도에서 불리할 수 있음

## 10. 시스템 구성 초안

## 12. 기대 효과

- 학생이 추상적인 기구학 개념을 직관적으로 이해할 수 있다.
- Windows 기반 수업 환경에서도 바로 활용할 수 있다.
- 로봇공학 교육용 오픈소스 도구로 발전 가능성이 있다.
- 경진대회에서 기술성, 교육성, 실용성을 동시에 보여줄 수 있다.

## 13. 결론

본 프로젝트는 `산업용 로봇 시뮬레이터의 축소판`이 아니라, `기구학 교육에 최적화된 Windows 기반 학습 도구`를 만드는 것을 목표로 한다.

초기 버전은 6축 매니퓰레이터와 DH 파라미터 기반 정기구학 시각화에 집중하고, 이후 역기구학과 과제 기능으로 확장한다.

직접 시뮬레이터를 구현하되, 물리엔진 전체가 아니라 교육적으로 중요한 계산과 시각화를 직접 설계하는 방향으로 범위를 통제한다.
